# 토스 차트 매매 UX 스펙

## 목표

토스 API로 가져온 캔들/현재가/계좌 정보를 한 화면에서 보고, 사용자가 과거 매수·매도 위치를 차트에서 복기하면서 주문 전 검증까지 진행할 수 있게 합니다.

## 사용자 맥락

- 토스 API 키를 등록한 로그인 사용자
- 자동매매 베타 사용자는 전략 체결 기록도 함께 확인
- 실거래는 기본 OFF이며, 주문 전 검증과 수동 확인을 먼저 통과

## 진입/이탈

- 진입: 포트폴리오 보유 종목, 자동매매 전략 상세, 종목분석 화면의 `차트 매매` CTA
- 이탈: `종목분석으로`, `포트폴리오로`, 주문/체결 기록 상세

## 화면 상태

- 미연결: 토스 키 등록 CTA, 공개 차트만 표시
- 연결됨: 차트, 포지션, 주문 패널, 체결 마커 표시
- 실거래 OFF: 주문 버튼은 `주문 검증`으로 표시
- 실거래 ON: 권한과 서버 킬스위치가 켜진 경우에만 실제 제출 버튼 표시
- 데이터 없음: 체결 마커 없는 빈 기록 상태 표시
- 오류: 토스 API 실패, rate limit, 장 마감, 주문 불가 사유 표시

## 핵심 동작

- 차트 마커 클릭: 체결가, 수량, 전략명, 손익 표시
- 기록 리스트 `보기`: 해당 마커 위치로 포커스
- 매수/매도 탭 전환: 주문 검증 버튼과 가능 금액/수량 기준 변경
- 주문 검증: 매수 가능 금액, 매도 가능 수량, 수수료, 상하한가, 장 상태 확인

## 정보 우선순위

1. 종목/현재가/장 상태
2. 차트와 과거 매매 마커
3. 보유수량/평단/손익
4. 주문 전 검증 패널
5. 체결/미체결/취소 기록

## 피드백 규칙

- 매수 마커는 녹색, 매도 마커는 빨간색
- 실거래 OFF는 상단 배지와 주문 버튼 문구로 항상 노출
- 주문 차단은 버튼 비활성보다 차단 사유를 먼저 표시
- API rate limit은 자동 재시도 시간과 함께 표시

## 수용 체크리스트

- 데스크톱에서 차트가 화면의 주 영역을 차지한다.
- 모바일에서는 차트가 먼저 나오고 주문 패널은 아래로 내려간다.
- 체결 마커와 기록 리스트가 같은 데이터를 가리킨다.
- 실거래 OFF 상태에서 실제 주문 제출처럼 보이지 않는다.
- 주문 API 호출은 서버의 `OrderIntent + RiskCheck + BrokerCredential + user ownership` 경계를 통과한다.

## 열린 질문

- 첫 버전에서 1분봉 차트를 기본으로 둘지, 일봉 + 현재가 polling으로 둘지 결정이 필요합니다.
- 체결 기록 보존 기간을 무제한으로 둘지, 사용자별 용량 제한을 둘지 정해야 합니다.
