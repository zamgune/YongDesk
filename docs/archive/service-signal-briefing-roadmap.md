# 서비스형 신호·브리핑·포트폴리오 개선 로드맵

> Archive · 2026-07-10: 완료됐거나 후속 방향이 바뀐 웹 중심 로드맵을 보존한 문서다. 현재 기능과 우선순위는 [기능 상태](../features.md)와 [이어서 개발하기](../continuation-guide.md)를 우선한다.

이 문서는 StockAnalysis를 단순 분석 도구에서 반복 사용 가능한 트레이딩 의사결정 서비스로 확장하기 위한 3단계 개선 목표와 완료 조건을 정리합니다.

핵심 순서는 다음과 같습니다.

```text
1단계: 신호 신뢰도 개선
→ 2단계: 데일리 리포트 고도화
→ 3단계: 내 포트폴리오 기준 오늘 할 일
```

세 단계는 독립 기능처럼 보이지만 실제로는 의존 관계가 있습니다. 신호 신뢰도가 먼저 있어야 데일리 리포트 후보 정렬이 의미를 갖고, 데일리 리포트 기준이 안정되어야 보유 종목별 오늘 할 일을 과하게 단정하지 않고 제시할 수 있습니다.

## 구현 상태

- 1단계 신호 신뢰도 개선: 구현 완료
  - `signalReliability` 계산 로직을 추가하고 단건 분석 API, 자동 대장주 스캔, 종목분석 UI에 연결했습니다.
  - 과거 유사 신호 수, 성공 비율, 손절 도달 비율, 평균 최대 상승/하락, 평균 손익비를 표시합니다.
- 2단계 데일리 리포트 고도화: 구현 완료
  - 데일리 리포트 후보 정렬에 신호 신뢰도를 반영했습니다.
  - `오늘의 돌파 후보`, `지지 확인 후보`, `주의 종목`, `스캔 상세 후보`가 같은 후보 점수와 신뢰도 정보를 공유합니다.
- 3단계 내 포트폴리오 기준 오늘 할 일: 구현 완료
  - 포트폴리오 탭에 보유 종목별 오늘 할 일 패널을 추가했습니다.
  - 데일리 브리핑 탭에서는 보유 종목이 있을 때 `내 포트폴리오 요약`을 표시합니다.

검증 기준:

- `npm run lint`
- `npm run build`
- `npm run test:market-briefing`
- `npm run test:signal-reliability`
- `npm run test:portfolio-daily-action`
- `/api/market/NVDA?days=365&tf=1d`
- `/api/market/005930.KS?days=365&tf=1d`
- `/api/briefing/daily-market?session=US&force=1`
- `/api/briefing/daily-market?session=KR&force=1`

## 공통 원칙

- 자동 매수/매도 지시가 아니라 의사결정 보조로 표현합니다.
- `매수`, `무조건 진입`, `승률 보장` 같은 단정 문구를 피합니다.
- 사용자에게 필요한 출력은 `현재 상태`, `진입 조건`, `실패 조건`, `주의점`, `다음 확인 시점`입니다.
- 신호 결과와 주문 실행은 분리합니다. 추후 자동매매가 붙더라도 `OrderIntent`와 `RiskCheck`를 반드시 거칩니다.
- 새 기능은 `domain → ports → use-cases → API/UI → tests/docs` 순서로 추가합니다.
- API Route와 UI 컴포넌트에 전략 계산 로직을 직접 누적하지 않습니다.

## 1단계: 신호 신뢰도 개선

### 목표

현재 종목 분석과 스캔 후보가 보여주는 돌파, 추세, 지지 확인 신호에 대해 “이 신호가 얼마나 믿을 만한지”를 정량화합니다.

v1의 목표는 완벽한 승률 예측이 아니라, 현재 조건과 유사한 과거 구간의 결과를 요약해 사용자가 신호를 더 현실적으로 판단하게 만드는 것입니다.

### 핵심 질문

- 이 종목에서 비슷한 돌파 신호가 과거에 몇 번 있었는가?
- 유사 신호 이후 목표 수익 구간까지 간 비율은 어느 정도인가?
- 신호 이후 손절선에 먼저 닿은 경우는 어느 정도인가?
- 평균 최대 상승률과 평균 최대 하락률은 어느 정도인가?
- 현재 신호는 손익비 관점에서 진입 가능한가?

### 구현 범위

- `SignalReliability` 도메인 타입 추가
- 돌파 신호, 추세 신호, 지지 확인 신호별 과거 유사 사례 탐색
- 과거 성과 요약 계산
  - 유사 신호 개수
  - 성공 비율
  - 평균 최대 상승률
  - 평균 최대 하락률
  - 손절 도달 비율
  - 평균 보유 기간
- 신뢰도 등급 표시
  - `높음`
  - `보통`
  - `낮음`
  - `데이터 부족`
- `/api/market/[symbol]` 응답에 신뢰도 블록 추가
- 종목분석 UI에 `신호 신뢰도` 패널 추가

### 체크리스트

- [ ] 신뢰도 계산 입력이 캔들 데이터와 기존 신호 결과로 제한되어 있다.
- [ ] look-ahead bias가 없도록 과거 신호 평가 시 미래 데이터 참조 범위가 명확하다.
- [ ] 유사 신호 개수가 부족하면 신뢰도를 높게 표시하지 않는다.
- [ ] 신뢰도 등급은 승률만 보지 않고 손익비와 손절 도달률을 함께 본다.
- [ ] 신고가 돌파, 박스권 돌파, 삼각수렴 돌파, 20일선 회복 신호를 구분한다.
- [ ] API 응답은 내부 계산값과 UI 표시 문구를 분리한다.
- [ ] 한국어 설명은 조건부 문장으로 작성한다.
- [ ] 기존 추세추종, 돌파 룰, 차트 품질 계산을 제거하지 않는다.

### 완료 조건

- [ ] `/api/market/NVDA?days=365&tf=1d` 응답에 `signalReliability`가 포함된다.
- [ ] `/api/market/005930.KS?days=365&tf=1d` 응답에 `signalReliability`가 포함된다.
- [ ] 종목분석 화면에서 신호 신뢰도 등급과 근거가 표시된다.
- [ ] 유사 신호가 부족한 종목은 `데이터 부족` 또는 `낮음`으로 표시된다.
- [ ] `npm run lint`가 새 오류 없이 통과한다.
- [ ] `npm run build`가 통과한다.
- [ ] 관련 테스트가 최소 1개 이상 추가된다.

## 2단계: 데일리 리포트 고도화

### 목표

데일리 브리핑을 단순 후보 나열이 아니라 “오늘 어떤 종목과 섹터를 우선 봐야 하는지”를 정리하는 서비스형 리포트로 개선합니다.

1단계에서 만든 신호 신뢰도를 사용해 후보를 정렬하고, 강한 종목과 위험한 종목을 더 명확히 구분합니다.

### 핵심 질문

- 오늘 시장에서 주도 섹터는 무엇인가?
- 해당 섹터 안에서 가장 강한 종목은 무엇인가?
- 돌파가 이미 확인된 후보와 아직 지지 확인이 필요한 후보는 무엇인가?
- 거래량이 동반된 돌파인지, 단순 가격 상승인지 구분되는가?
- 오늘 신규 진입을 검토할 후보와 관찰만 해야 할 후보가 분리되는가?

### 구현 범위

- 데일리 리포트 섹션 재정리
  - 시장 상태 요약
  - 오늘의 주도 테마
  - 강한 돌파 후보
  - 지지 확인 후보
  - 주의 종목
  - 스캔 상세 후보
- 후보 정렬에 신호 신뢰도 반영
- 후보 카드에 신뢰도, 돌파 상태, 거래량 확인, 실패선 표시
- `왜 오늘 봐야 하는지` 문구 추가
- `데일리 나스닥`, `데일리 한국장`의 출력 구조 통일
- `scanCandidates`와 리포트 상단 후보가 같은 점수 기준을 공유하도록 정리

### 체크리스트

- [ ] 데일리 리포트 후보가 단순 50일 수익률만으로 정렬되지 않는다.
- [ ] 신호 신뢰도, 돌파 상태, 거래량 확인, 차트 품질이 함께 반영된다.
- [ ] 주도 테마와 강한 종목의 연결 관계가 UI에서 보인다.
- [ ] `confirmed`, `retest`, `watch`, `failed` 상태가 명확히 구분된다.
- [ ] 신규 진입 후보와 관찰 후보를 같은 톤으로 섞어 보여주지 않는다.
- [ ] 미국장과 한국장 리포트가 같은 정보 구조를 사용한다.
- [ ] 장외거래는 미국장 보조 정보로만 표시하고 정규장 확정 신호로 쓰지 않는다.
- [ ] 모바일에서 후보 카드가 겹치거나 잘리지 않는다.

### 완료 조건

- [ ] `/api/briefing/daily-market?session=US` 응답에 고도화된 후보 섹션이 포함된다.
- [ ] `/api/briefing/daily-market?session=KR` 응답에 고도화된 후보 섹션이 포함된다.
- [ ] 데일리 브리핑 UI에서 `강한 돌파 후보`, `지지 확인 후보`, `주의 종목`을 확인할 수 있다.
- [ ] 후보 카드 클릭 시 기존처럼 종목분석으로 이동한다.
- [ ] `npm run test:market-briefing`이 통과한다.
- [ ] `npm run lint`가 새 오류 없이 통과한다.
- [ ] `npm run build`가 통과한다.
- [ ] README 또는 현재 기능 문서가 새 리포트 구조를 반영한다.

## 3단계: 내 포트폴리오 기준 오늘 할 일

### 목표

사용자가 보유 중인 종목과 관심 종목을 기준으로 오늘 확인해야 할 행동 후보를 요약합니다.

이 단계의 핵심은 “전체 시장에서 좋은 종목”이 아니라 “내가 가진 종목을 오늘 어떻게 관리할지”를 알려주는 것입니다.

### 핵심 질문

- 보유 종목 중 손절선에 가까워진 종목은 무엇인가?
- 수익권에서 20일선 추적 모드로 전환된 종목은 무엇인가?
- 추가매수를 검토할 수 있는 종목은 무엇인가?
- 신규 진입은 가능하지만 아직 지지 확인이 필요한 종목은 무엇인가?
- 오늘 아무 행동을 하지 않는 것이 더 나은 종목은 무엇인가?

### 구현 범위

- `PortfolioDailyAction` 도메인 타입 추가
- 보유 종목별 오늘 할 일 계산
  - `보유 유지`
  - `지지 확인`
  - `추가매수 대기`
  - `수익 실현 검토`
  - `손절선 근접`
  - `신규 진입 보류`
  - `데이터 부족`
- 평단가, 현재가, 손절가, 20일선, 돌파 기준선, 신호 신뢰도 반영
- 포트폴리오 탭에 `오늘 할 일` 패널 추가
- 데일리 브리핑에 `내 포트폴리오 요약` 섹션 추가
- 브라우저 localStorage 기반 포트폴리오를 유지하되, 추후 서버 저장으로 전환 가능한 입력/출력 구조로 설계

### 체크리스트

- [ ] 포트폴리오 액션은 단정적 주문 지시가 아니라 검토 기준으로 표현된다.
- [ ] 보유 종목과 미보유 관심종목의 문구가 분리된다.
- [ ] 평단가가 있는 경우 수익률과 리스크 기준에 반영된다.
- [ ] 평단가가 없는 경우 신규 진입 기준 중심으로 표시된다.
- [ ] 손절선 근접, 수익 실현 검토, 추가매수 대기가 동시에 충돌하지 않도록 우선순위가 있다.
- [ ] 신호 신뢰도가 낮거나 데이터가 부족하면 공격적인 액션을 표시하지 않는다.
- [ ] 모바일 포트폴리오 카드에서 오늘 할 일 문구가 잘리지 않는다.
- [ ] 추후 서버 저장 전환을 고려해 계산 로직이 UI localStorage에 직접 묶이지 않는다.

### 완료 조건

- [ ] 포트폴리오 탭에서 보유 종목별 오늘 할 일이 표시된다.
- [ ] 데일리 브리핑에서 내 포트폴리오 기준 요약이 표시된다.
- [ ] 수익권 종목은 20일선 추적 또는 수익 실현 검토 기준이 표시된다.
- [ ] 손실권 종목은 손절선 근접 여부와 실패 조건이 표시된다.
- [ ] 추가매수 후보는 5일선/20일선/돌파 기준선 중 어떤 기준을 보는지 표시된다.
- [ ] `npm run lint`가 새 오류 없이 통과한다.
- [ ] `npm run build`가 통과한다.
- [ ] 포트폴리오 관련 테스트 또는 use-case 테스트가 최소 1개 이상 추가된다.

## 이번 로드맵에서 제외하는 것

- 실제 자동 주문 실행
- 증권사 API 연결
- 회원가입과 서버 DB 저장
- 승률 보장형 문구
- 전체 시장 모든 종목 실시간 스캔
- 유료 결제와 권한 관리

이 항목들은 서비스화에 중요하지만, 먼저 신호 품질과 사용자 의사결정 흐름이 안정된 뒤 추가하는 것이 안전합니다.
