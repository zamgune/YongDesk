# 신고가 돌파 보조 룰

## 목적

신고가 돌파 보조 룰은 Threads식 전략에서 참고한 `신고가 돌파`, `강한 종목 선별`, `-10% 손절`, `+20% 이후 20일선 추적` 원칙을 기존 StockAnalysis 추세추종 판단 위에 얹은 보조 분석입니다.

이 룰은 자동 매수/매도 지시가 아닙니다. 기존 5일선/20일선/거래량/대장주 필터를 제거하지 않고, 강한 종목을 읽는 추가 근거로만 사용합니다.

## 계산 기준

- 신고가 기준가: 최근 120봉 또는 252봉의 이전 고점
- 신고가 거리: 현재가가 신고가 기준가 대비 얼마나 위/아래에 있는지
- 거래대금 강도: 최근 20봉 평균 `close * volume`
- 돌파/지지 거래량: 최신 봉 거래량을 직전 20봉 평균 거래량과 비교한 배율
- 고정 손절: 신고가 기준가 대비 -10%
- 수익 추적 전환: 신고가 기준가 대비 +20%
- 추적 이탈선: 20일선
- 돌파 지지선: 신고가 기준가 또는 이전 고점 저항선
- 실패선: 돌파 지지선 아래 여유 구간과 구조 손절 중 더 보수적인 기준

## 손절/분할익절 관리

돌파매매 판단은 진입 가능 여부만 보여주지 않고, 진입 이후 무효화와 수익 보호 기준을 함께 제시합니다.

- 돌파매매 손절: `breakoutSignal.failureLevel` 또는 `tradeSetup.failureLevel`을 우선 사용합니다.
- 전체 손절: 최근 저점 기반 강제 손절, 신고가 기준 -10% 보조 손절, 셋업 실패선 중 최종 방어선을 표시합니다.
- 추적 손절: +20% 수익 추적 전환 이후에는 20일선 또는 `trailingExitPrice`를 잔여 물량 기준으로 표시합니다.
- 분할 손절: 셋업 손절 이탈 시 30%, 추적/주요 지지선 이탈 시 추가 40%, 전체 손절 이탈 시 잔여 30% 정리를 검토합니다.
- 분할익절: 저항선 또는 1R에서 30%, 2R 또는 +20% 전환가에서 추가 30%, 잔여 40%는 추적 손절 전까지 추세 보유로 봅니다.
- 손실 중 비중 확대는 기본 전략에 포함하지 않고, 기준선 회복과 거래량 재확인 이후 재진입 조건으로만 다룹니다.

## 상태값

```text
breakout-ready: 신고가 돌파 후보입니다. 거래량과 지지 확인이 필요합니다.
wait-pullback: 추세 구조는 살아 있지만 아직 신고가 돌파가 확인되지 않았습니다.
profit-tracking: +20% 수익 추적 전환 이후 20일선 추적 모드입니다.
risk-off: 20일선 또는 -10% 보조 손절 기준을 위협합니다.
avoid: 신고가 돌파와 20일선 구조가 모두 부족합니다.
```

## API 필드

`/api/market/[symbol]`, `/api/market/leaders`, `/api/market/auto-leaders`는 `breakoutRule` 블록을 반환할 수 있습니다.

```text
status
newHighLevel
breakoutDistancePct
avgTradedValue20
volumeConfirmation
fixedStopPrice
profitSwitchPrice
trailingExitPrice
reasons[]
```

`avgTradedValue20`은 평소 유동성 확인용이고, `volumeConfirmation`은 돌파 또는 지지 확인 시점에 거래량이 붙었는지를 보는 수급 확인용입니다.

```text
volumeConfirmation.ratio20: 최신 봉 거래량 / 직전 20봉 평균 거래량
volumeConfirmation.status: strong | confirmed | normal | weak | unavailable
volumeConfirmation.context: breakout | support | none
```

신고가 돌파는 5일선/20일선 눌림만으로 해석하지 않습니다. 돌파 직후에는 이전 고점 저항선이 지지선으로 바뀌는지가 핵심입니다. 따라서 `tradeSetup.type === "breakout"`이면 `keyLevelLabel`은 `돌파 지지선`이고, 일봉 종가가 `failureLevel` 아래에서 마감할 때 돌파 실패로 해석합니다.

공통 `tradeSetup` 블록:

```text
type: breakout | pullback | reclaim | extended | risk-off
keyLevel: 핵심 기준가
keyLevelLabel: 돌파 지지선 | 5일선 | 20일선 | 회복 기준선 | 20일선 추적선 | 방어 기준선
failureLevel: 기준선 실패 가격
validIf: 기준선 유효 조건
invalidIf: 기준선 실패 조건
entryPlan: 진입 또는 추가매수 방식
stopReason: 실패 기준의 이유
```

## 화면 반영

- `종목분석`: 상단 판단은 `최종 판단 -> 진입 조건 -> 실패 조건 -> 차트 -> 상세 지표` 순서로 표시합니다.
- `종목분석`: `돌파매매 판단`은 박스권, 삼각수렴, 컵앤핸들, 신고가, 20일선 회복 후보 중 가장 강한 셋업을 기준으로 `진입 가능`, `분할 가능`, `확인 대기`, `리테스트 대기`, `추격 주의`, `제외`로 분류합니다.
- `종목분석`: `진입 가능`은 돌파 상태가 확인되고, 차트 품질이 약하지 않으며, 거래량 확인이 동반된 경우에만 표시합니다.
- `종목분석`: `분할 가능`은 돌파 신호와 거래량은 좋지만 이격 또는 손절폭 부담이 있어 풀진입 대신 1차 탐색 비중만 검토하는 상태입니다.
- `종목분석`: `손절/분할익절 기준` 패널은 돌파매매 손절, 전체 손절, 1차/2차 분할익절, 분할 손절 비중, 재진입 조건을 우선 표시합니다.
- `종목분석`: 신고가 기준가, 현재가 대비 거리, 거래대금, 돌파/지지 거래량, -10% 손절가, +20% 전환가, 20일선 추적선은 상세 기준 영역에서 보조 정보로 표시합니다.
- `시장 스캔`: 후보 카드에 신고가 돌파, 거래량 확인, +20% 추적, 20일선 이탈 주의 배지를 표시합니다.
- `시장 스캔`: 후보 카드에 `돌파 확인`, `지지 재확인`, `패턴 대기`, `돌파 실패` 같은 돌파 상태 배지를 표시합니다.
- `데일리 브리핑`: 강한 후보 목록에 신고가 돌파 상태, 거래량 확인 상태, 기준선/실패선을 함께 표시합니다.
- `데일리 브리핑`: `오늘의 돌파 후보` 섹션에서 돌파 상태와 차트 품질 점수로 후보를 정렬합니다.
- `포트폴리오`: 보유 카드에 1차 익절, 2차 익절, 추적 손절, 최종 손절 가격을 표시하고, 펼침 영역에서 30/30/40 분할익절과 30/40/30 분할 손절 기준을 보여줍니다.
- `포트폴리오`: +20% 이상 수익권이거나 `profit-tracking` 상태이면 `20일선 추적 모드` 배지를 표시하고, 손절/익절 판단에도 같은 기준선 언어를 사용합니다.
- 미국장 장외 보조 체크: 장외가가 신고가 기준가 위에 있어도 정규장 돌파 확정으로 보지 않고 참고 문구로만 표시합니다.

## 문구 원칙

- `신고가 돌파 즉시 매수`처럼 단정하지 않습니다.
- `진입 가능`은 조건 충족 후보라는 뜻이며, 자동 매수 또는 무조건 진입 지시가 아닙니다.
- 신규 진입은 `돌파 후보`, `분할 가능`, `지지 확인`, `추격 주의`로 표현합니다.
- 보유 관리는 `20일선 추적`, `수익 보호`, `리스크 관리`로 표현합니다.
- `불타기` 대신 `추가매수`, `익절` 대신 `수익 실현`을 사용합니다.
